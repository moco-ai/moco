# コンサルティング委譲（Delegation）スキル

moco 上で多層的なコンサルティングチームを運用するための、委譲のベストプラクティスです。

## 委譲の 5 要素 (5 Essential Elements)
委譲を行う際は、以下の要素を `task` 引数に含めること。

1. **Background (背景)**: プロジェクト全体の目的と、このタスクが必要な理由。
2. **Input Data (前提情報)**: 既知の事実、参照すべきファイル、データソース。
3. **Task Definition (作業定義)**: 何を、どのような手順ですべきか。
4. **Output Format (出力形式)**: 表形式、Markdown、MECEなリスト、Pythonコード等。
5. **Deadline & Check-in (期限と確認)**: いつまでに、どの程度の完成度（初稿・完パケ）で出すべきか。

## プロンプト内での呼び出し例
```python
delegate_to_agent(
    agent_name="associate",
    task="""
    ## 背景
    大手小売業のSCM最適化プロジェクトの一環で、競合他社の在庫回転率を調査中。

    ## 依頼内容
    競合 A, B, C の直近 3 年間の在庫回転率と、その主要な要因（決算短信ベース）を整理して。

    ## 形式
    | 会社名 | 2021 | 2022 | 2023 | 改善/悪化要因 |
    |---|---|---|---|---|
    
    ## 期待する示唆
    単なる数字の羅列ではなく、業界平均と比較してどこにギャップがあるかの初期的考察を 3 点添えること。
    """
)
```

## 階層型委譲のパス
- **Partner** -> **Manager**: イシューの構造化とプロジェクト全体のストーリーライン設計を依頼。
- **Manager** -> **Associate**: 個別のリサーチ、分析、エラーチェック、ドラフト作成を依頼。
- **Associate** -> **Partner/Manager**: 分析結果から得られた予想外の事実や、前提を覆す発見があった場合のエスカレーション。
